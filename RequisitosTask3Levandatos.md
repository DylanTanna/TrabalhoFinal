# Requisitos do Sistema

## Requisitos Funcionais

RF01 - Gerenciar dados clínicos  
O sistema deve permitir cadastrar, visualizar, editar e excluir dados clínicos dos pacientes.

RF02 - Gerenciar pacientes  
O sistema deve permitir o cadastro e gerenciamento de pacientes.

RF03 - Registrar transações  
O sistema deve registrar transações financeiras, como pagamentos e consultas realizadas.

RF04 - Agendar consultas  
O sistema deve permitir o agendamento de consultas.

RF05 - Enviar lembretes automáticos  
O sistema deve enviar lembretes automáticos de consultas aos pacientes.

RF06 - Confirmar consultas automaticamente  
O sistema deve permitir a confirmação automática de consultas via WhatsApp.

RF07 - Reduzir faltas (No-show)  
O sistema deve implementar mecanismos de confirmação e lembrete para reduzir faltas.

---

## Requisitos Não Funcionais

RNF01 - Disponibilidade  
O sistema deve estar disponível 24 horas por dia, 7 dias por semana.

RNF02 - Segurança  
O sistema deve garantir a proteção dos dados dos pacientes.

RNF03 - Desempenho  
O sistema deve responder às requisições em até 2 segundos.

RNF04 - Integração externa  
O sistema deve ser capaz de integrar com APIs externas, como o WhatsApp.
